# Baseline Critique Behavior Analysis

## What the Agent Does Naturally (Without Skill)

### Strengths Observed
1. **Identifies obvious problems** - market saturation, competition, technical challenges
2. **Provides structured feedback** - organized into clear sections
3. **Suggests alternatives** - offers different directions
4. **Asks clarifying questions** - tries to understand better
5. **Challenges assumptions** - questions the core premise

### Gaps and Weaknesses

#### 1. Stops at Surface Level
- Identifies "trust problem" but doesn't explore trust mechanics deeply
- Mentions "network effects" without quantifying switching costs
- Notes "semantic preservation" but doesn't push into edge cases
- Lists "code smells" without exploring measurement frameworks

#### 2. Conservative Depth
- Gives 3-4 alternative directions but doesn't push each to extremes
- Suggests "niche focus" but doesn't force specificity (how narrow is narrow enough?)
- Mentions risks but doesn't scenario-plan catastrophic failures
- Provides critique but doesn't force iterative refinement

#### 3. Settles for "Good Enough" Analysis
- **Rationalization**: "This is thorough enough for initial feedback"
- **Reality**: Innovation requires exhaustive exploration, not satisfactory coverage
- Stops after 10-12 critique points when there could be 30+
- Doesn't cross-pollinate ideas (what if we combined X approach with Y from different domain?)

#### 4. Doesn't Challenge Fundamental Assumptions
- Accepts the problem statement as given
- Doesn't question: "Is this even the right problem to solve?"
- Doesn't push: "What if the opposite were true?"
- Missing: "What would have to be true for this to be revolutionary?"

#### 5. Linear Thinking
- Follows predictable structure: problem → analysis → alternatives
- Doesn't employ lateral thinking techniques
- Doesn't use analogies from other domains
- Doesn't explore absurd extremes to find insights

#### 6. Incomplete Innovation Dimensions

**Not explored:**
- Second-order effects (what happens when this succeeds?)
- Emergence potential (what new behaviors might emerge?)
- Combinatorial possibilities (what could this enable that doesn't exist?)
- Paradigm shifts (what assumptions does this break?)
- 10x vs 10% thinking (could this be 10x better, not incrementally better?)

#### 7. Validation Gaps
- Suggests validation but doesn't design specific experiments
- Doesn't propose falsifiable hypotheses
- Doesn't identify minimum viable tests
- Doesn't create decision trees for "kill vs pivot vs proceed"

## Specific Rationalizations Observed

| What Agent Did | What Innovation Requires |
|----------------|-------------------------|
| "Here's a thorough critique" | "Here are 50 unexplored angles" |
| "Consider these alternatives" | "Let's exhaust every alternative until we find the breakthrough" |
| "This might work if..." | "Let's prove this can't work, then find why that proof is wrong" |
| Listed 5-6 challenges | Map every possible failure mode systematically |
| Asked clarifying questions | Challenge every assumption including the question itself |
| Provided structured feedback | Unstructured exploration until novel insight emerges |

## Missing Techniques

1. **Inversion** - What if we did the opposite?
2. **First Principles** - What are the atomic truths here?
3. **Constraint Removal** - What if X limitation didn't exist?
4. **Constraint Addition** - What if we added limitation Y?
5. **Adjacent Possible** - What's one step beyond current capability?
6. **Combinatorial Innovation** - What if we combined with Z from different field?
7. **Extreme Scenarios** - Best case? Worst case? Weirdest case?
8. **Meta-Analysis** - What does the shape of this problem tell us?

## What "No Stone Unturned" Actually Means

The agent stops when feedback "feels complete." Innovation requires:
- Exhausting every angle even when it feels redundant
- Pushing past "that's probably not relevant" instincts
- Exploring connections that seem far-fetched
- Testing assumptions that seem obviously true
- Continuing after "good enough" is reached

## Bottom Line

**Natural behavior: Helpful, structured, competent critique**
**Innovation requirement: Relentless, exhaustive, uncomfortable deep-diving**

The skill needs to counteract the agent's tendency toward:
- Satisfactory completion
- Linear thinking
- Conservative depth
- Stopping at "thorough enough"
