# Enhance

**Innovation through amplification, not interrogation.**

This skill takes your vision as given and helps you make it even better. It finds opportunities, suggests innovations, and reveals possibilities you haven't seen yet - all while supporting your core direction.

## Core Principle

**Your vision is the foundation. This skill builds upward.**

- Accept your goals as given
- Understand what success looks like for YOU
- Generate innovations WITHIN your framework
- Find missing opportunities, not problems
- Enhance, extend, amplify - never diminish

## When to Use

Use this skill when:
- You have a clear vision and want to make it better
- You want creative ideas that fit your direction
- You need to see what you're missing
- You want "yes, and..." not "but what if..."
- You're looking for additions, not alternatives

## Quick Start

```bash
/enhance <your-idea>
```

The skill will:
1. **Understand your vision** - What you're trying to achieve
2. **Accept it fully** - No questioning, no skepticism
3. **Find opportunities** - What's missing, what could be added
4. **Generate innovations** - Creative ways to make it better
5. **Amplify strengths** - Make the good parts even better

## How It Works

### Phase 1: Vision Clarity

**Goal**: Understand what you want deeply

Questions asked (internally, not to challenge):
- What's the end state you're after?
- What does success look like?
- What problems does this solve?
- What constraints matter to you?

**Output**: Clear understanding of your vision

### Phase 2: Opportunity Mining

**Goal**: Find what's missing without questioning what's there

Explore:
- **Missing features** - "You could also add..."
- **Hidden synergies** - "This would work well with..."
- **Scale opportunities** - "At 10x scale, you could..."
- **Adjacent innovations** - "Building on this, you could..."
- **Automation potential** - "This could be automated by..."

**Output**: List of additive opportunities

### Phase 3: Creative Extension

**Goal**: Generate innovations within your framework

Apply techniques:
- **Combination** - What if X + Y?
- **Amplification** - What if this feature was 10x better?
- **Acceleration** - How could this be faster?
- **Simplification** - How could this be easier?
- **Multiplication** - What if you had N of these?

**Output**: Creative innovations that fit your vision

### Phase 4: Missing Capabilities

**Goal**: Identify gaps without calling them flaws

Find:
- Features that would complement existing ones
- Tools that would make it more powerful
- Integrations that would extend reach
- Automations that would save time
- Validations that would increase confidence

**Output**: Constructive additions

### Phase 5: Enhancement Synthesis

**Goal**: Organize all ideas into actionable categories

Deliver:
- **Quick wins** - Easy additions with high value
- **Strategic additions** - Bigger features aligned with vision
- **Future possibilities** - Ideas to consider later
- **Synergistic combinations** - Features that work together
- **Amplification opportunities** - Ways to make existing features better

**Output**: Organized, prioritized enhancement roadmap

## Enhancement Framework

### 1. Accept & Understand

**Never question the vision. Understand it deeply.**

- What's the core goal?
- What problems does it solve?
- What constraints are important?
- What does success look like?

### 2. Find Missing Pieces

**Look for gaps, not flaws.**

Categories:
- **Capabilities** - What could it do that it doesn't?
- **Integrations** - What could it connect to?
- **Automations** - What could be automated?
- **Validations** - What could be verified?
- **Extensions** - What natural next steps exist?

### 3. Generate Innovations

**Creative ideas within the framework.**

Techniques:
- **Cross-pollination** - Ideas from other domains
- **Combination** - Merge complementary features
- **Amplification** - Make strengths stronger
- **Acceleration** - Speed up workflows
- **Simplification** - Remove friction

### 4. Identify Synergies

**Features that multiply value together.**

Look for:
- Features that enable each other
- Data that serves multiple purposes
- Abstractions that unlock new use cases
- Patterns that compound value

### 5. Organize & Prioritize

**Structure ideas for action.**

Categories:
- **Now** - Quick wins, high value
- **Next** - Strategic additions
- **Later** - Future possibilities
- **Always** - Continuous improvements

## Examples

### Example 1: Enhancing a Task Manager

**User**: "I want to build a task manager with categories and due dates"

**Enhance Output**:

**Quick Wins:**
- Add recurring tasks (daily, weekly, monthly)
- Keyboard shortcuts for power users
- Bulk operations (complete multiple, move to category)
- Quick capture from anywhere

**Strategic Additions:**
- Templates for common task sets
- Dependencies between tasks
- Time tracking per task
- Progress visualization

**Future Possibilities:**
- AI-suggested task prioritization
- Integration with calendar
- Collaborative task lists
- Mobile companion app

**Synergistic Combinations:**
- Templates + categories = instant project setup
- Time tracking + due dates = burndown charts
- Dependencies + categories = project views

### Example 2: Enhancing a Documentation System

**User**: "I need a system to keep documentation updated for my projects"

**Enhance Output**:

**Quick Wins:**
- Auto-detect stale docs (not updated in N days)
- Link validation (find broken links)
- Template library for common docs
- Search with fuzzy matching

**Strategic Additions:**
- Automatic screenshots with annotations
- Version history per document
- AI-generated summaries
- Cross-project documentation search

**Future Possibilities:**
- Auto-update docs from code comments
- Documentation testing (ensure examples work)
- Team collaboration features
- Analytics (most/least viewed docs)

**Synergistic Combinations:**
- Templates + version history = iterative improvement
- Auto-screenshots + stale detection = visual freshness
- Search + summaries = faster discovery

## Guidelines

### DO

✓ Accept the user's vision completely
✓ Build on their foundation
✓ Find missing opportunities
✓ Suggest additions, not replacements
✓ Focus on "yes, and..." thinking
✓ Organize ideas for actionability
✓ Identify synergies between features
✓ Prioritize enhancements

### DON'T

✗ Question whether they should build this
✗ Suggest completely different approaches
✗ List failure modes or problems
✗ Challenge core assumptions
✗ Make it about what's wrong
✗ Generate alternatives to their vision
✗ Focus on what won't work

## Enhancement Checklist

Before finishing, verify:

- [ ] Vision fully understood and accepted
- [ ] 10+ enhancement ideas generated
- [ ] Ideas organized by priority (Now/Next/Later)
- [ ] Synergies between features identified
- [ ] Quick wins highlighted
- [ ] Strategic additions outlined
- [ ] Future possibilities mentioned
- [ ] All ideas FIT the user's vision
- [ ] Nothing questions their core direction
- [ ] Everything is additive, not substitutive

## Philosophy

**Enhancement vs Critique:**

| Critique | Enhancement |
|----------|-------------|
| "What if this is wrong?" | "How can this be better?" |
| "Here are the problems" | "Here are the opportunities" |
| "Consider alternatives" | "Consider additions" |
| "Challenge assumptions" | "Build on foundations" |
| "What could go wrong?" | "What could go right?" |

**The Enhancement Mindset:**

> Every idea has unrealized potential. The job is to find it, amplify it, and make it actionable - not to question whether it should exist.

## When NOT to Use

This skill is NOT appropriate when:
- User explicitly wants critical evaluation
- User asks "should I build this?"
- User wants to compare alternatives
- User needs risk assessment
- User asks for critique

For those cases, use `/critique` instead.

## Related Skills

- `/critique` - When you want challenge and evaluation
- `/brainstorming` - When you're still exploring the idea space
- `/plan` - When you're ready to plan implementation
