# Enhance Skill

**Make your ideas better, not different.**

## What This Skill Does

The `/enhance` skill is a generative collaborator that:
- Accepts your vision completely
- Finds opportunities you haven't seen
- Suggests additions and improvements
- Organizes enhancements by priority
- Identifies synergies between features

## When to Use

Use `/enhance` when you:
- Have a clear vision and want to expand it
- Need creative ideas that fit your direction
- Want "yes, and..." thinking, not "but what if..."
- Are looking for additions, not alternatives
- Want supportive innovation, not critical evaluation

## Quick Start

```bash
/enhance <your-idea>
```

Example:
```bash
/enhance I want to build a task manager with categories and due dates
```

## What You Get

The skill delivers:

1. **Quick Wins** - Easy additions with high value
2. **Strategic Enhancements** - Bigger features aligned with vision
3. **Future Possibilities** - Ideas to consider later
4. **Synergistic Combinations** - Features that multiply value together
5. **Missing Capabilities** - Gaps that would make it more complete

## Tips

**Get the most from /enhance:**

1. **Be specific about your vision** - "I want X because Y"
2. **Mention constraints** - "Must be simple" or "Should scale to 1M users"
3. **State priorities** - "Speed is more important than features"
4. **Ask for specific focus** - "Focus on user experience enhancements"

**Better prompts:**
- "Enhance my API design focusing on developer experience"
- "Make this authentication flow more secure without adding complexity"
- "Improve this data pipeline for better observability"

**Less effective:**
- "Make it better" (too vague)
- "What should I add?" (not specific)

## When NOT to Use

Don't use /enhance when you:
- Want critical evaluation
- Need to compare alternatives
- Want risk assessment
- Are questioning whether to build it

For those cases, use `/critique` instead.
