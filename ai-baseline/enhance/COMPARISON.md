# Enhance vs Critique: When to Use Each

## Quick Decision Guide

**Use /enhance when:**
- ✓ You have a clear vision
- ✓ You want to make it better
- ✓ You need supportive innovation
- ✓ You want "yes, and..." thinking
- ✓ You're looking for additions

**Use /critique when:**
- ✓ You're unsure about direction
- ✓ You want challenge and evaluation
- ✓ You need risk assessment
- ✓ You want alternative approaches
- ✓ You're open to questioning assumptions

## Side-by-Side Comparison

| Aspect | /enhance | /critique |
|--------|----------|-----------|
| **Stance** | Supportive collaborator | Critical evaluator |
| **Goal** | Make idea better | Evaluate idea validity |
| **Questions** | "How can this be better?" | "Should this exist?" |
| **Focus** | Opportunities | Problems and risks |
| **Output** | Additions & enhancements | Alternatives & challenges |
| **Tone** | "Yes, and..." | "But what if..." |
| **Assumptions** | Accepted as given | Questioned and tested |
| **Vision** | Amplified | Challenged |

## Example: Same Idea, Different Skills

**Idea:** "Build a task manager with categories and due dates"

### /enhance Output:
```
Quick Wins:
- Add recurring tasks
- Keyboard shortcuts
- Bulk operations
- Quick capture widget

Strategic Enhancements:
- Task templates
- Dependencies
- Time tracking
- Progress visualization

This builds on your solid foundation!
```

### /critique Output:
```
Challenges:
- Market is saturated with task managers
- What's the unique differentiation?
- Have you validated users want another one?
- Consider: Is task management the real problem?

Alternatives:
- Focus on specific niche (developers? students?)
- Solve upstream problem (why tasks pile up)
- Build anti-productivity tool instead

This questions whether to build it at all.
```

## The Right Tool for the Job

**"I know what I want to build"** → /enhance
- Adds features you haven't thought of
- Finds synergies between ideas
- Organizes enhancements by priority
- Makes your vision even stronger

**"I'm exploring what to build"** → /critique
- Tests your assumptions
- Presents alternatives
- Identifies failure modes
- Helps validate direction

## When You're Unsure Which to Use

Ask yourself:

**"Am I open to NOT building this?"**
- YES → Use /critique
- NO → Use /enhance

**"Do I want to defend my idea or expand it?"**
- Defend → Use /critique (it will test you)
- Expand → Use /enhance (it will help)

**"Am I in exploration or execution mode?"**
- Exploration → Use /critique
- Execution → Use /enhance

## Can You Use Both?

Absolutely! Use them in sequence:

### Pattern 1: Critique Then Enhance
```
1. /critique <initial-idea>
2. Evaluate alternatives, pick direction
3. /enhance <chosen-direction>
4. Get implementation enhancements
```

Best for: New ideas you're still validating

### Pattern 2: Enhance Then Critique
```
1. /enhance <your-vision>
2. Get comprehensive feature set
3. /critique <enhanced-version>
4. Identify risks and tradeoffs
```

Best for: Strong vision that needs stress-testing

### Pattern 3: Iterative
```
1. /enhance → Get features
2. /critique → Test assumptions
3. /enhance → Improve based on feedback
4. Repeat until confident
```

Best for: Complex projects needing both support and challenge

## Philosophy

**Enhancement is about building UP.**
Your idea is the foundation. We add floors, rooms, features.

**Critique is about testing DOWN.**
Your idea is the hypothesis. We test, challenge, validate.

Both are valuable. Choose based on what you need right now.

---

**Remember:**
- /enhance believes in your vision and helps you realize it
- /critique questions your vision and helps you validate it

Neither is better. They serve different purposes.
